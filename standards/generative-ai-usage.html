Prompts containing PII or confidential data MUST NOT be sent to external LLM providers without approved data loss prevention controls.

Sending PII or confidential data to external LLM providers risks data leakage, regulatory non-compliance, and violation of customer data protection agreements.

Verification: DLP scanning is enforced at the API gateway layer for all outbound LLM requests; audit logs are reviewed for policy violations.

RAG patterns provide up-to-date, auditable responses grounded in enterprise data without the cost and complexity of fine-tuning, while maintaining clear data provenance.

Verification: Architecture review confirms RAG implementation with approved vector databases; response attribution to source documents is validated.

Guidelines

GUIDE-001

Prompt engineering best practices: Use system prompts to establish clear behavioral boundaries. Implement few-shot examples for consistent output formatting. Apply chain-of-thought prompting for complex reasoning tasks. Version control all prompt templates alongside application code.

GUIDE-002

RAG architecture should use a chunking strategy appropriate to the content type (e.g., semantic chunking for documents, fixed-size for code). Implement hybrid search combining dense vector similarity with sparse keyword matching. Maintain metadata for source attribution and freshness tracking.

GUIDE-003

Cost optimization strategies: Implement prompt caching for repeated queries. Use smaller models for classification and routing tasks before escalating to larger models. Set maximum token limits per request. Batch non-real-time requests during off-peak hours for lower pricing tiers.

GUIDE-004

Red teaming and adversarial testing should be conducted before any GenAI application is deployed to production. Test for prompt injection, jailbreaking, data extraction attacks, and hallucination rates. Document findings and remediation actions in the application security review.

Definitions

TermDefinition
Retrieval-Augmented Generation (RAG) A pattern that enhances LLM responses by retrieving relevant documents from an external knowledge base and including them as context in the prompt, grounding outputs in factual enterprise data.
Guardrails Safety mechanisms implemented around LLM inputs and outputs to filter harmful content, enforce topic boundaries, prevent data leakage, and ensure responses comply with organizational policies.
Prompt Injection An adversarial attack where malicious instructions are embedded in user input to manipulate an LLM into ignoring its system prompt, bypassing safety controls, or revealing confidential information.
Token The fundamental unit of text processing for LLMs. A token typically represents 3-4 characters of English text. Token counts determine both the context window limits and the cost of LLM API calls.

Frequently Asked Questions

Can we use public LLM APIs like ChatGPT for internal business applications?
Public LLM APIs may only be used for non-sensitive, non-confidential use cases after completing a risk assessment. For any workload involving PII, proprietary data, or customer information, you must use approved private endpoints (Azure OpenAI Service or AWS Bedrock) with enterprise data protection controls.
What is the process for getting a new GenAI use case approved?
Submit a GenAI Use Case Proposal through the AI Center of Excellence. The proposal must include a data classification assessment, risk evaluation, cost estimate, guardrail implementation plan, and human oversight design. The AI Review Board evaluates proposals within 10 business days.
How do we handle hallucinations in customer-facing GenAI applications?
Implement RAG patterns to ground responses in verified enterprise data. Add confidence scoring and citation of source documents. Use output validation rules to detect and filter unsupported claims. For high-stakes interactions, implement human-in-the-loop review before responses are delivered to customers.
